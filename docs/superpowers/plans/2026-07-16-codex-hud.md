# Codex HUD Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [x]`) syntax for tracking.

**Goal:** Build a Codex-native counterpart to Claude HUD that persistently displays project, model, context, quota, tool, MCP, skill, subagent, task, and session activity below Codex CLI.

**Architecture:** A TypeScript CLI starts the official `codex` binary inside a tmux session and renders a sibling HUD pane without patching Codex. A rollout adapter incrementally reads `$CODEX_HOME/sessions/**/rollout-*.jsonl`, a config adapter reads Codex and Hub configuration, and a pure renderer reproduces Claude HUD's compact/expanded layouts. A repo-local Codex plugin bundles setup/configuration skills and lifecycle metadata while the executable remains independently usable.

**Tech Stack:** Node.js 20+, TypeScript, pnpm 10, tsdown, Vitest, `@antfu/eslint-config`, tmux, Codex rollout JSONL, Codex plugin manifest.

**Completion note (2026-07-16):** Implemented and verified against Claude HUD 0.5.0. Codex-only telemetry boundaries and the unsupported plugin-hook manifest field are documented in `docs/claude-hud-parity.md`. Final verification: 42 tests, lint, typecheck, build, publint, packed-artifact audit, three skill validators, plugin validator, managed install, cached-plugin doctor, single-session smoke, concurrent same-cwd isolation, and natural tmux cleanup.

## File map

- `package.json`: npm metadata, binaries, scripts, supported Node version.
- `pnpm-workspace.yaml`: named dependency catalogs.
- `tsconfig.json`, `tsdown.config.ts`, `eslint.config.mjs`, `vitest.config.ts`: build and quality tooling.
- `src/types/*.ts`: config, rollout protocol, normalized HUD state, render types.
- `src/config/*.ts`: defaults, validation, persistence, presets, interactive configuration.
- `src/codex/*.ts`: Codex home/config discovery, session selection, incremental rollout parsing, rate-limit normalization.
- `src/collectors/*.ts`: Git, project, memory, installed plugins/MCP, process/session metadata.
- `src/render/*.ts`: ANSI colors, width handling, individual HUD lines, compact/expanded composition.
- `src/runtime/*.ts`: tmux launcher, renderer loop, resize/key handling, cleanup, diagnostics.
- `src/cli.ts`: command routing for `start`, `render`, `configure`, `doctor`, `install`, `uninstall`.
- `plugins/codex-hud/`: standard Codex plugin manifest, skills, hooks, scripts, and assets.
- `.agents/plugins/marketplace.json`: repo-local marketplace entry.
- `tests/fixtures/*.jsonl`: redacted protocol fixtures for active, completed, compacted, quota, and subagent sessions.
- `README.md`, `README.zh.md`, `LICENSE`, `NOTICE`: usage, compatibility, attribution, and licensing.
